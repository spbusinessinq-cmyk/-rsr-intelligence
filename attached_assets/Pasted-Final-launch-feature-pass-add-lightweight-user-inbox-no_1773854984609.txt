==================================================
1. USER INBOX / NOTIFICATIONS SYSTEM
==================================================

Add a simple user notifications system.

Requirements:
- create/use a Supabase table for notifications
- notifications should belong to users
- each notification should support at least:
  - id
  - user_id
  - title
  - body
  - type
  - link (optional)
  - is_read
  - created_at

==================================================
2. USER EXPERIENCE
==================================================

Users need a clear way to see notifications.

Requirements:
- add a small inbox / notifications entry point in the UI
- show unread count if there are unread items
- provide a clean list view / panel
- allow notification to be marked read
- allow mark all read if easy
- clicking a notification should go to its linked destination if link exists

Keep styling fully consistent with current site aesthetic.

==================================================
3. ADMIN / SYSTEM CREATION
==================================================

Add lightweight admin notification controls.

Requirements:
- from Command Console, allow admin to create/send notifications

- no fake actions
- notifications must persist to Supabase

Do NOT overbuild beyond this.
